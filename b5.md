## PHẦN 1: PHÂN TÍCH & THIẾT KẾ LUỒNG NGHIỆP VỤ (SYSTEM DESIGN)

### 1.1. Xác định Input / Output
* **Input (Đầu vào):**
  * `userId`: Định danh của người dùng đang đăng nhập (hoặc ẩn danh nếu chưa đăng nhập).
  * `currentProductId`: Định danh sản phẩm người dùng đang xem ở giao diện chi tiết (Product Detail Page).
  * `purchaseHistory`: Lịch sử các đơn hàng đã thanh toán của người dùng (lấy từ Database).
  * `userBehaviorLog`: Nhật ký hành vi click, thêm vào giỏ hàng, hoặc tìm kiếm gần đây (lấy từ Cache/Event Log).
* **Output (Đầu ra):**
  * `recommendationList`: Danh sách chứa tối đa 10 đối tượng sản phẩm gợi ý (bao gồm: ID, tên, giá, hình ảnh, điểm số tương đồng/gợi ý) xếp hạng theo độ phù hợp giảm dần.

### 1.2. Sơ đồ Luồng Logic Nghiệp Vụ (ASCII Sequence Workflow)
```text
[User Client]          [API Gateway]         [Recommendation Service]       [Database / Cache]
      |                      |                          |                          |
      |-- 1. View Product -->|                          |                          |
      |   (userId, prodId)   |-- 2. Forward Request --->|                          |
      |                      |                          |-- 3. Query History ----->|
      |                      |                          |      & Behavior Logs     |
      |                      |                          |<-- 4. Return User Data --|
      |                      |                          |                          |
      |                      |                          |-- 5. Calculate Weights --|
      |                      |                          |   (Collaborative/Content)|
      |                      |                          |                          |
      |                      |                          |-- 6. Filter Out of Stock|
      |                      |                          |      & Purchased Items  |
      |                      |                          |                          |
      |                      |<-- 7. Return List -------|                          |
      |                      |   (Top 10 Products)      |                          |
      |<-- 8. Render UI -----|                          |                          |
```

## PHẦN 3: MINH CHỨNG THỰC TẾ (TEXT LOG PHIÊN TƯƠNG TÁC LLM)

*Dưới đây là bản sao chép log dữ liệu thực tế kết quả phản hồi sau khi chạy chuỗi Prompt trên hệ thống:*

### [TEXT LOG BƯỚC 2.1] - Kết quả sinh Use Case & Functional Requirements

```markdown
# ĐẶC TẢ YÊU CẦU CHỨC NĂNG - MODULE AI PRODUCT RECOMMENDATION

## 1. MÔ TẢ USE CASE (USE CASE DESCRIPTION)

- **Tên Use Case:** Nhận gợi ý sản phẩm thông minh (Get Smart Product Recommendations)
- **Tác nhân (Actors):** Người dùng (User/Customer), Hệ thống AI Engine.
- **Điều kiện tiên quyết (Pre-conditions):** Người dùng đang truy cập vào trang chi tiết của một sản phẩm bất kỳ (`currentProductId`).
- **Điều kiện sau khi hoàn thành (Post-conditions):** Hệ thống hiển thị danh sách tối đa 10 sản phẩm liên quan có tỷ lệ chuyển đổi cao nhất lên giao diện.

### Luồng xử lý chính (Main Flow):
1. Người dùng click xem trang chi tiết sản phẩm mã số X.
2. Hệ thống Front-end gửi request kèm theo định danh người dùng (`userId`) và mã sản phẩm hiện tại (`currentProductId`) về Backend.
3. Hệ thống Backend truy vấn lịch sử mua hàng (`purchaseHistory`) và danh mục của sản phẩm X để làm dữ liệu đầu vào cho mô hình lọc cộng tác (Collaborative Filtering) và lọc dựa trên nội dung (Content-based Filtering).
4. Hệ thống AI Engine tính toán điểm số trọng số (Recommendation Score) của các sản phẩm liên quan trong kho.
5. Hệ thống lọc bỏ các sản phẩm đã hết hàng hoặc sản phẩm chính người dùng đang xem.
6. Hệ thống trả về danh sách Top 10 sản phẩm có điểm số cao nhất.
7. Giao diện Front-end render khối nội dung "Sản phẩm gợi ý dành riêng cho bạn" ở phía dưới trang chi tiết.

### Luồng rẽ nhánh & Ngoại lệ (Alternative Flows):
- **AL-01: Người dùng mới (Cold Start Problem):** Nếu người dùng chưa đăng nhập hoặc là tài khoản mới hoàn toàn chưa có lịch sử mua hàng, hệ thống sẽ tự động chuyển sang luồng gợi ý dựa trên: Top các sản phẩm được mua nhiều nhất (Best Sellers) thuộc cùng danh mục của sản phẩm X.
- **AL-02: Kho hàng trống:** Nếu các sản phẩm được gợi ý có số lượng tồn kho bằng 0, hệ thống tự động loại bỏ và bù vào bằng các sản phẩm có điểm số tương đồng kế tiếp cho đến khi đủ 10 sản phẩm.

---

## 2. YÊU CẦU CHỨC NĂNG (FUNCTIONAL REQUIREMENTS)

- **FR-REC-01:** Hệ thống phải tự động nhận diện trạng thái đăng nhập của người dùng để quyết định luồng thuật toán (Cá nhân hóa hoặc Cold Start).
- **FR-REC-02:** Backend phải cung cấp API RESTful trả về danh sách gợi ý dạng JSON trong thời gian trễ cho phép.
- **FR-REC-03:** Thuật toán AI phải cập nhật trọng số gợi ý dựa trên danh mục (Category), nhãn hiệu (Brand), và khoảng giá (Price Range) của sản phẩm hiện tại để đảm bảo tính "liên quan".
- **FR-REC-04:** Hệ thống phải lọc bỏ các sản phẩm trùng lặp hoặc sản phẩm đang bị ẩn/ngừng kinh doanh ra khỏi danh sách kết quả trước khi phản hồi về Client.
```

### [TEXT LOG BƯỚC 2.2] - Kết quả sinh Từ điển dữ liệu (Data Dictionary)

```markdown
## 3. TỪ ĐIỂN DỮ LIỆU SƠ BỘ (DATA DICTIONARY)

### 3.1. Cấu trúc dữ liệu đầu vào (Request Parameters / Payload)
- **Endpoint:** `GET /api/v1/recommendations`
- **Định dạng:** Query Parameters

| Field Name | Data Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| `product_id` | Long | Mắt buộc (Required) | ID của sản phẩm người dùng đang xem tại trang chi tiết. |
| `user_id` | Long | Tùy chọn (Optional) | ID của người dùng (nếu đã đăng nhập). Truyền `null` nếu là khách vãng lai. |
| `limit` | Integer | Tùy chọn (Mặc định: 10) | Số lượng sản phẩm tối đa muốn lấy về để hiển thị trên UI. Giới hạn tối đa $\le 20$. |

### 3.2. Cấu trúc dữ liệu đầu ra (Response JSON Payload)
- **Định dạng:** JSON Object (Array of Objects)

| Field Name | Data Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| `recommended_products` | Array | Không null | Danh sách mảng chứa các đối tượng sản phẩm được gợi ý. |
| `id` | Long | Bắt buộc | ID duy nhất của sản phẩm được gợi ý. |
| `product_name` | String | Bắt buộc | Tên hiển thị của sản phẩm. |
| `thumbnail_url` | String | Định dạng URL | Đường dẫn tuyệt đối tới ảnh đại diện thu nhỏ của sản phẩm. |
| `price` | BigDecimal | Giá trị $\ge 0$ | Giá bán hiện tại của sản phẩm (đã áp dụng giảm giá nếu có). |
| `original_price`| BigDecimal | Giá trị $\ge 0$ | Giá gốc chưa giảm của sản phẩm để Front-end hiển thị gạch ngang. |
| `match_score` | Double | Khoảng [0.00 - 1.00] | Điểm số độ tương đồng/phù hợp do thuật toán AI tính toán. |
```
